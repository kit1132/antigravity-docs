# 記事執筆ガイドライン

このフォルダ（`note/`）配下で記事を作成する際は、以下のスタイルと構成に従ってください。

## 文体とトーン

- **親しみやすく、かつプロフェッショナルに**: 読者に語りかけるような、しかし信頼感のあるトーン
- **です・ます調**: 基本は丁寧語を使用する
- **強調の活用**: **太字**で重要なポイントを視覚的に強調する
- **簡潔さ**: 冗長な表現を避け、要点を明確に伝える

## 記事の構成

以下の順序で構成することを推奨します。

1. **タイトル**: 魅力的で内容を端的に表すもの
2. **はじめに**: 問題提起や背景、読者の共感を呼ぶ導入
3. **結論**: 記事の核となるメッセージを早期に提示（結論ファースト）
4. **メリット/理由**: リスト形式や見出しを使って詳細に説明
5. **具体的な手順/方法**: ステップバイステップで分かりやすく
6. **まとめ/アクション**: 読者が次に何をすべきかを提示

## フォーマット規則

### 見出し

- `#` : 記事タイトル（1つのみ）
- `##` : 大セクション
- `###` : サブセクション

### リスト

- 箇条書きは `-` を使用
- 番号付きリストは手順説明に使用

### コードブロック

- コマンドや設定例は ``` で囲む
- 言語指定がある場合は ```bash のように指定

### 図表

- ASCII図やテーブルで視覚的に情報を整理
- 複雑な概念は図解で補足

## 記事の長さ目安

- **短い記事**: 1,000〜2,000字（Tips、小ネタ）
- **標準記事**: 3,000〜5,000字（解説、チュートリアル）
- **長編記事**: 5,000字以上（包括的ガイド）
